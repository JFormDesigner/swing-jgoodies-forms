What are the main benefits of the JGoodies Forms?

The JGoodies Forms aims to make the layout process faster, better and easier. It has been designed to make simple things easy and the hard stuff possible, good design easy and the bad difficult.

Can I use the Forms in a commercial product?

You can use, copy and redistribute the JGoodies Forms in source and binary form - even for commercial use - if you comply with the very relaxed terms of the BSD open source license. The BSD license has been chosen, because it allows many uses, and the few license terms are short and human readable, even if you're not a lawyer.

How to get started?

First, open and read the README.html and check the available documentation: the RELEASE-NOTES, Tutorial, Reference, API Docs (JavaDocs), Tips & Tricks, and Pointers to Visual Builders and other online resources. Then study the Advanced Layout presentation or the JGoodies Forms Whitepaper; both are available at the JGoodies articles page.

The Forms tutorial is an important part of this distribution. It demonstrates different ways to use the Forms library, and some typical pitfalls. The tutorial sources ship in folder src/tutorial. It is strongly recommended to download and run the JGoodies Forms Demo. This demo contains many Forms tutorial examples and will aquaint you with the library features.

Where do I get the latest version?

The latest version is available at java.net in the project's download section. There you can get the library, sources, tutorial, documentation, the Forms whitepaper, and the Forms Demo.

Where do I get older versions?

Archived versions are available at java.net in the project's Archive download folder. The Forms version 1.0.2 supports Java 1.3.

Where do I get support?

Support is available only in the user mailing list users@forms.dev.java.net. Before you post, check this FAQ, the relevant JavaDocs, and the accompanying tutorial sources. You can find and track the progress of Forms issues in the issue tracker. As a project observer you can log defects, request features and suggest enhancements.

The user mailing list helps us reduce the costs for managing this freeware project. In the mailing list all project users can jump in to assist you, if you face a problem with the library or API. And other users can benefit from your questions and our answers. Personal mail to JGoodies.com bypasses the other project developers, and it often leads to unnecessary extra work; it slows down the development process, and it costs time and money on our side.

If you have downloaded the library at no charge, I ask you to register with the user mailing list before you post, and post only with the registered account. Otherwise your messages won't show up automatically, and I need to manually accept every message.

Priority support is available for commercial licensees of the JGoodies Swing Suite. These can contact us by phone or direct questions to support at JGoodies dot com.

If you have general questions about Swing, layout and design, please post to the Java Desktop forums at www.java.net. That's a great place to share information with the Java desktop community. And we frequently check this forum and participate in discussions.

How to report a problem?

Before you report a problem, check this FAQ, the relevant JavaDocs, and the accompanying tutorial sources.

If you think you've found a bug in the JGoodies Forms, please check the issue tracker or ask in the user mailing list, if this issue is already known.

If you report a bug, please provide at least the following information: JRE vendor, JRE version, JRE build, OS, OS version, Forms version, and whether the problem can be reproduced with the JGoodies Forms Demo or not. For example: Sun Java 1.5.0_06, Windows XP, Validation 1.1.1, can be reproduced with the JGoodies Forms Demo. Please point us to the Forms Demo, if it shows a problem. If you cannot reproduce the problem with the Forms Demo, please describe the steps to produce the problem. Often a screenshot makes it easier to understand a problem, or a request for an enhancement.
